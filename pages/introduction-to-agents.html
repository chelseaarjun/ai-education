Introduction to Agents

Understanding AI Agents: From Concepts to Applications

Welcome to this module on AI Agents! In this lesson, you'll explore how AI agents are transforming the way we interact with AI systems, enabling more powerful and autonomous applications.

AI agents represent the next evolution in artificial intelligence, moving beyond passive models to active systems that can perceive, reason, and take actions to achieve specific goals. By the end of this module, you'll understand the fundamentals of AI agents, their key components, and how they can be applied to solve real-world problems.

Fundamentals of AI Agents

What is an AI Agent?

An agent is an LLM-powered system designed to take actions and solve complex tasks autonomously. Unlike traditional LLMs, AI agents go beyond simple text generation. They are equipped with additional capabilities, including:

  • Tools: They can interact with external tools and resources, such as databases, APIs, and software applications, to gather information and execute actions.
  • Memory: AI agents can store and retrieve information, allowing them to learn from past experiences and make more informed decisions.
  • Planning: AI agents can analyze a problem, break it down into steps, and adjust their approach based on new information.
Analogy: Think of a restaurant scenario. With an LLM, you're limited to ordering items exactly as they appear on the menu. An agent, however, is like asking a chef to create a vegan dish in a non-vegan restaurant - it can assess available ingredients, find a suitable recipe, and prepare a delicious meal that wasn't on the menu. The agent actively solves the problem by combining resources and knowledge in new ways.

From LLMs to Agents: The Key Differences

LLMs Agents
Passive text generators Active decision-makers
Respond to requests without memory Maintain context across exchanges
Process → Predict next token Process → Plan → Act → Observe → Repeat
Limited to text generation Use tools and APIs to affect the real world
Confined to training data Access and act on external information

The LLM as the Agent's Brain: The foundation model (LLM) serves as the "brain" of an agent. Models with stronger reasoning capabilities excel at powering agents:

  • OpenAI: GPT-4o and GPT-4 Turbo are optimized for tool use and planning
  • Anthropic: Claude 3.5 Sonnet and Claude 3 Opus feature enhanced reasoning
  • Google: Gemini 1.5 Pro and Ultra support complex reasoning chains

These models better understand context, follow complex instructions, and utilize tools effectively. The agentic behavior emerges from this foundation combined with external frameworks for perception, memory, and tool use.

The Agent Cycle: Core Functions

Every agent operates through a continuous cycle:

Let's examine each step:

  • Process: The agent gathers and interprets information from various sources
    • User inputs, data feeds, API responses, memory retrieval
    • Example: Understanding a math problem and its requirements
  • Plan: The agent determines the best course of action
    • Evaluating options, planning steps, prioritizing actions
    • Example: Choosing the appropriate formula and calculation steps
  • Act: The agent executes its decisions
    • Using tools, generating outputs, modifying data
    • Example: Performing the calculation and formatting the result
  • Observe: The agent assesses the outcomes of its actions
    • Evaluating results, detecting changes, gathering feedback
    • Example: Verifying the solution makes sense in context

This cycle repeats as the agent works toward its goals, with each iteration potentially bringing it closer to completion.

Simple Math Agent Example

Let's see a math problem-solving agent in action, going through multiple cycles:

from langchain.agents import AgentExecutor, create_react_agent from langchain_core.tools import Tool from langchain_bedrock import Bedrock from langchain_core.prompts import ChatPromptTemplate # Initialize Claude Sonnet on Amazon Bedrock llm = Bedrock( model_id="anthropic.claude-3-sonnet-20240229-v1:0", region_name="us-west-2", model_kwargs={"temperature": 0} ) # Define tools the agent can use tools = [ Tool( name="Calculator", func=lambda x: eval(x), description="Useful for performing calculations" ), Tool( name="UnitConverter", func=lambda x: f"Converted {x}", # Simplified placeholder function description="Converts between different units of measurement. Format: 'value from_unit to_to_unit'" ) ] # Create a prompt template for the agent prompt = ChatPromptTemplate.from_messages([ ("system", "You are a helpful math assistant that can solve problems step-by-step."), ("human", "{input}") ]) # Initialize the agent agent = create_react_agent(llm, tools, prompt) agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True) # Example interaction result = agent_executor.invoke({ "input": "A store has a rectangular shelf that is 2m long and 60cm deep. If I have boxes that are 20cm x 15cm x 10cm, how many boxes can I place on the shelf in a single layer?" }) print(result["output"])

Let's trace how this agent works through just 4 cycles:

First Cycle:

  • Process: Agent receives the problem about placing boxes on a rectangular shelf
  • Plan: Notices mixed units (m and cm) and decides to convert all measurements to the same unit
  • Act: Uses UnitConverter to convert shelf length from meters to centimeters: 2m → 200cm
  • Observe: Records that the shelf dimensions are now 200cm x 60cm

Second Cycle:

  • Process: Agent now has all dimensions in consistent units (centimeters)
  • Plan: Recognizes this is a single-layer arrangement, so only the length and width of the boxes matter
  • Act: Identifies that boxes can be placed in two orientations:
    • Option 1: 20cm x 15cm footprint
    • Option 2: 15cm x 20cm footprint
  • Observe: Decides to check both orientations to maximize the number of boxes

Third Cycle:

  • Process: Agent analyzes the first orientation (20cm x 15cm)
  • Plan: Calculates how many boxes fit along each dimension
  • Act: Uses Calculator to compute:
    • Along length: 200 ÷ 20 = 10 boxes
    • Along depth: 60 ÷ 15 = 4 boxes
    • Total: 10 × 4 = 40 boxes
  • Observe: Records that 40 boxes fit with the first orientation

Fourth Cycle:

  • Process: Agent analyzes the second orientation (15cm x 20cm)
  • Plan: Calculates how many boxes fit along each dimension in this orientation
  • Act: Uses Calculator to compute:
    • Along length: 200 ÷ 15 = 13.33 (13 whole boxes)
    • Along depth: 60 ÷ 20 = 3 boxes
    • Total: 13 × 3 = 39 boxes
  • Observe: Compares both arrangements and provides the final answer that 40 boxes can fit using the first orientation (20cm x 15cm)

This example shows how the agent repeatedly cycles through the Process → Plan → Act → Observe loop, with each cycle building on information from previous cycles.

What is the primary difference between an LLM and an AI agent?
  • a) LLMs are less advanced than agents
  • b) Agents actively take actions and use tools to achieve goals
  • c) LLMs cannot understand human language
  • d) Agents do not use language models
Explanation: The key difference is that agents are active systems that can take actions and use tools to accomplish goals, while LLMs are primarily focused on generating text responses to prompts.

  • b) It processes requests faster
  • c) It has enhanced reasoning capabilities for planning and tool use
  • d) It costs less to operate

  • a) Process
  • b) Plan
  • c) Act
  • d) Observe
