Introduction to Prompt Engineering

Welcome to this guide on prompt engineering! Today, you'll explore how to effectively communicate with artificial intelligence models like Claude and Amazon Bedrock to get the best possible results for your applications.

Prompt engineering is a crucial skill in the AI era. By the end of this 30-minute lesson, you'll understand how to craft effective prompts that can help you build sophisticated AI applications, even without extensive programming knowledge.

What are Prompts?

A prompt is the input you provide to an AI system to elicit a specific output. Think of it as the interface between human intent and AI capability—they're how we communicate what we want the model to do.

In technical terms, a prompt is a sequence of tokens (words, characters, or subwords) that provides context and instructions to a language model.

Simple Prompt: "What is machine learning?"
More Detailed Prompt: "Explain machine learning to a high school student in 3 paragraphs, covering supervised learning, unsupervised learning, and reinforcement learning."

Why Prompt Engineering Matters

  • Precision: Well-crafted prompts yield more accurate and useful outputs
  • Efficiency: Better prompts reduce iterations and token usage, saving time and costs
  • Consistency: Systematic prompting leads to more predictable results
  • Capability Unlocking: Many advanced AI capabilities are accessible only through proper prompting
Which of these is NOT a benefit of effective prompt engineering?
  • 1. Reduced computational requirements for the model
  • 2. More accurate responses
  • 3. Fewer iterations to get desired results
  • 4. Better predictability of outputs
Explanation: While good prompts may save tokens, they don't change the computational needs of the underlying model. Better prompts can improve accuracy, reduce the number of iterations, and make outputs more predictable, but they don't reduce the fundamental computational requirements of the AI model itself.

The Prompt Engineering Mindset

Successful prompt engineers think from both perspectives:

  • As the human: What exact information or action do I need?
  • As the AI model: What instructions and context will help it understand my intent?

This dual perspective helps bridge the gap between human expectations and how AI systems actually process information.

Anatomy of an Effective Prompt

Example Basic Prompt Structure: [INSTRUCTION]: Create a summary of the following customer feedback [CONTEXT]: This feedback is from a user of our mobile banking app [INPUT]: "The app keeps crashing when I try to deposit checks using my camera. Otherwise it's pretty good and I like the new transfer feature." [OUTPUT FORMAT]: Provide a 2-sentence summary followed by bullet points for key issues and one positive aspect.

Core Prompting Principles

  • Start simple and add complexity only as needed
  • Iterate and refine - treat prompt engineering as an experimental process. Document, measure, iterate
  • Structure matters - organize your prompt with clear components (instructions first or last, context, examples, format)
  • One task at a time - break complex requests into sequential prompts
  • Leverage role-playing - assign the AI a role for context-appropriate answers.
  • Be Specific about the output - specify the exact tone, style, max length and output format you want.
  • Show, don't just tell - use mix of examples to demonstrate the desired output format and style
  • Use step-by-step reasoning - encourage reasoning and to process in sequential steps, especially for complex tasks.
  • Be clear and percise - use unambiguous terms, remove unnecessary context or instructions.
  • Use templates - reuse prompts and make it more dynamic by managing prompt as templates with dynamic variables.

Vague prompt: Tell me about data structures.
Precise prompt: Explain three fundamental data structures (arrays, linked lists, and hash tables) used in software engineering. For each one, provide: 1. A one-sentence definition 2. Its primary strengths and weaknesses 3. One common use case in modern applications

With practice, you'll develop an intuition for which approaches work best in different situations, allowing you to effectively harness the power of models like Claude and Amazon Bedrock for your applications.

Essential Prompt Design Techniques

Zero-shot, One-shot, and Few-shot Prompting

These techniques relate to the Input Examples component of a prompt structure and determine how many examples you include:

  • Zero-shot: No examples provided, just instructions
  • One-shot: A single example demonstrating the desired behavior
  • Few-shot: Multiple examples that establish a pattern
Zero-shot prompting asks the model to perform a task without examples:
Classify this review as positive, negative, or neutral: "The service was quick but the food was cold when it arrived."
One-shot prompting provides a single example:
Here's an example of classifying sentiment: Review: "The interface is confusing and buggy." Classification: Negative Now classify this review: "The service was quick but the food was cold when it arrived."
Few-shot prompting gives multiple examples:
Here are some examples of sentiment classification: Review: "The interface is confusing and buggy." Classification: Negative Review: "Amazing product, works exactly as advertised!" Classification: Positive Review: "Shipping was fast but packaging was minimal." Classification: Neutral Now classify this review: "The service was quick but the food was cold when it arrived."
When to use: Few-shot prompting is especially useful when you need the AI to recognize specific patterns or follow a particular format that might not be obvious from instructions alone.

Instruction Based Prompting

Modern AI systems often distinguish between different types of prompts in a conversation:

  • System Prompt: Sets persistent behavior patterns and guidelines for the entire conversation. A system prompt may specify a role for the AI to adopt (like "you are a teacher") or simply provide general instructions about response style, format, and constraints.
  • User Prompt: Contains the specific requests or queries from the human.
  • Assistant Response: The model's generated reply based on the system and user prompts.
Example with no system prompt:
User: Explain how Kubernetes pods communicate with each other within a cluster.
Example with basic system prompt:
System: Provide concise explanations with relevant examples. Use bullet points for lists.
User: Explain how Kubernetes pods communicate with each other within a cluster.
Example with role-specified system prompt:
System: You are a technical documentation specialist who excels at creating clear, concise explanations of complex topics. Always provide examples when explaining concepts.
User: Explain how Kubernetes pods communicate with each other within a cluster.

Effective system prompt instructions include:

  • Length limits: "Keep your response under 100 words"
  • Format requirements: "Structure your answer as a JSON"
  • Complexity levels: "Explain this at a 9th-grade reading level"
  • Specifying Roles: "Roles ask the AI to adopt a specific persona allowing for controlling expertise level, communication style, Domain Knowledge"

Constraint-based Prompting

A constraint is a set of limitations or boundaries on the response. It limits what the model should not do or avoid.

Without constraints:
Explain quantum computing.
With constraints:
Explain quantum computing to a high school student. Your explanation should: - Avoid mathematical formulas

As a best practice, start by prioritizing instructions, clearly stating what you want the model to do and only use constraints when necessary for safety, clarity or specific requirements.

Which prompting technique would be best for ensuring a specific output structure with categories and formatting?
  • 1. Zero-shot prompting
  • 2. Role-based prompting
  • 3. Format control prompting
  • 4. System prompting
Answer: 3. Format control prompting is specifically designed to ensure outputs follow a precise structure and organization.

Advanced Techniques

Chain-of-thought Prompting

This technique encourages the model to show its reasoning process step-by-step:

Standard prompt:
If a store has 120 apples and sells 30% on Monday, then sells 40% of the remaining apples on Tuesday, how many apples are left?
Chain-of-thought prompt:
Problem: If a store has 120 apples and sells 30% on Monday, then sells 40% of the remaining apples on Tuesday, how many apples are left? Solve this step-by-step, showing your work for each stage of the calculation.

Benefits include:

  • More accurate results for complex reasoning
  • Transparent decision-making
  • Easier error detection
  • Educational value

Prompt Chaining

Breaking complex tasks into a sequence of simpler prompts:

Implementation approaches:
1. Manual chaining (human in the loop):
First prompt: "Extract the key metrics from this quarterly report."
Human reviews results
Second prompt: "Based on these metrics, identify the three most concerning trends."
Human reviews results
Third prompt: "For each trend, recommend two specific actions the company could take."
2. Programmatic chaining (automated):
Using tools like LangChain or custom code to automatically pass the output of one prompt as input to the next, creating a chain of AI operations that can solve complex tasks step by step.
3. Hybrid approach:
Combination of automated chaining with human review at critical decision points, particularly useful for high-stakes applications where oversight is necessary.

Prompt chaining is especially powerful for:

  • Breaking down complex reasoning into manageable steps
  • Maintaining focus on a specific subtask at each stage
  • Enabling human review of intermediate results
  • Creating reusable prompt components for different workflows

ReAct (Reason + Act)

ReAct combines reasoning with actions in an iterative process:

You are helping debug a Python application. For each error: 1. THINK: Analyze what might be causing the error 2. ACT: Suggest a specific code change 3. OBSERVE: Consider what effect this change would have 4. DECIDE: Determine if this resolves the issue or if further changes are needed Error: "TypeError: cannot unpack non-iterable int object"
The ReAct pattern is especially useful for:
  • Complex problem-solving
  • Troubleshooting
  • Decision-making with multiple factors
  • Self-correcting processes

Retrieval-augmented Generation (RAG)

RAG enhances prompts with relevant information retrieved from external sources:

I'll provide information about our company's return policy, then ask you to answer a customer question based ONLY on this information. Policy: "Items can be returned within 30 days with receipt for full refund. After 30 days but before 60 days, store credit is offered. Electronics have a 15-day return window. Sale items are final sale." Customer question: "I bought a sweater on clearance three weeks ago, but it has a tear in the seam. Can I return it?"

Key components of a RAG system:

  • Document storage and indexing
  • Retrieval mechanism
  • Context integration
  • Response generation
Which prompting technique is best suited for multi-step problem solving that requires both reasoning and taking specific actions?
  • 1. Few-shot prompting
  • 2. Chain-of-thought prompting
  • 3. ReAct prompting
  • 4. RAG prompting
Answer: 3. ReAct (Reasoning + Acting) prompting specifically combines reasoning steps with action steps in an iterative process, making it ideal for complex problem solving.

Technique Selection Guide

    Technique When to Use
    Zero-shot Simple, direct queries with no need for examples
    Few-shot Tasks that benefit from format or style examples
    Chain-of-thought Complex reasoning, math, logic, or explanations needed
    Prompt Chaining Multi-step tasks requiring output from one prompt as input to the next
    ReAct Tasks that combine reasoning with action or tool-use (e.g., search, tools)
    RAG Tasks needing accurate, up-to-date, or domain-specific external information

Common Pitfalls and Solutions

Pitfall Description Impact Solution
Prompt Ambiguity Instructions open to multiple interpretations Model answers a different question than intended or provides incomplete responses Use specific verbs and clear success criteria
Overconstraining Too many requirements, creating conflicts Model struggles to satisfy all constraints, leading to suboptimal responses Prioritize constraints and remove unnecessary ones
Context Overload Providing too much information Key instructions get lost in noise or exceed context window Focus on relevant context only
Inadequate Priming Not establishing the right initial framework or mindset Model lacks necessary context to generate appropriate responses Begin with clear role and purpose statements
Hallucination Risk Not constraining the model to known information Model generates plausible but incorrect information Request citations or add phrases like "if you're unsure, say so" or "indicate your confidence level for each claim"

Measuring Prompt Effectiveness

Evaluate your prompts based on:

  1. Accuracy: Does the response correctly address the request?
  2. Completeness: Does it cover all aspects of what was asked?
  3. Relevance: Does it stay focused on the specific task?
  4. Efficiency: How many tokens/iterations were required?
  5. Consistency: Do similar prompts produce similarly good results?

For systematic evaluation, create a prompt testing framework:

  • Define benchmark tasks
  • Create evaluation criteria
  • Measure against baselines
  • Iterate based on results
Which prompting techniques would be most effective when you need to ensure a logical, step-by-step solution to a complex mathematical problem?
  • 1. Role-based + Format control
  • 2. Chain-of-thought + Constraint-based
  • 3. Few-shot + ReAct
  • 4. Zero-shot + RAG
Answer: 2. Chain-of-thought prompting encourages step-by-step reasoning, while constraint-based prompting helps focus the solution approach and format. These techniques work well together for complex mathematical problems.
